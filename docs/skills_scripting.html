General Scripting for *Nix Systems

Scripting in any *nix environment

The title of this section is actually a play on regular expression capabilities, meaning "match all text that ends in nix". So, this section applies to any Linux, Posix (Mac OS), or general bash environment (including Windows using Windows Subsystem for Linux or Git Bash).

Should your team wish to adjust the Jenkins pipeline or create small scripts for your use within this repository, you will need to know basic scripting languages available to you on the production system and on your local machine. If you are adjusting the Jenkins pipeline, please make sure you're familiar with how Jenkinsfiles work first.

Languages of Choice in a *nix environment

The best languages to use for scripting purposes in a *nix environment for this project specifically are any of the following:

  • The Bash Shell/Language homepage here.
    • Standard, no-frills, comprehensive introduction guide here.
    • Quick start introduction guide here.
    • Fun way to learn bash here.
  • Python 3
    • Do NOT use Python 2 for this reason.
    • Doing things that you can normally do in bash, in python here.
    • Extremely popular guide to automating stuff on your computer with python here.
  • Node.js (Since we're already using it for this project).
    • Replacing shell scripts with Node.js here.
    • Another good guide to shell scripting with Node.js here.

Warning/Caution about Bash alternatives

We do not recommend using alternative Bash/sh flavors such as Fish, zsh, etc. because we want to standardize development environments. Also, the default shell on the production system uses bash.
